# Customer Onboarding Checklist

## Required Information (must have before accepting first order)

- [ ] Full legal company name
- [ ] Primary contact name and email
- [ ] Physical business address (not PO Box only)
- [ ] Company phone number
- [ ] Industry/sector classification
- [ ] How they found Rev A

## Recommended Information (collect when available)

- [ ] Company website URL
- [ ] DBA/trade name (if different from legal)
- [ ] Shipping address (if different from HQ)
- [ ] Primary contact direct phone
- [ ] Primary contact title/role
- [ ] Approximate company size (employees)
- [ ] DUNS number
- [ ] Estimated annual spend potential
- [ ] BD rep assignment

## Legitimacy Verification (complete before first order)

- [ ] Website verification — confirm the company has a functioning website
- [ ] Email domain verification — contact email matches company domain
- [ ] Business presence — at least one independent confirmation (LinkedIn, industry directory, state registration, news mentions)
- [ ] Red flag screening — no indicators of fraud, competitors, or sanctioned entities
- [ ] Export control screening — no ITAR/EAR flags on the customer or their end use

## CRM Setup (complete before first order)

- [ ] Account record created in CRM
- [ ] Contact record created and linked to Account
- [ ] Account status set to "Onboarding"
- [ ] Source/origination recorded
- [ ] BD rep linked (if applicable)
- [ ] PM assigned and linked

## Credit Setup (complete before first order ships)

- [ ] Initial credit terms established
- [ ] Credit application sent (for orders > $5,000 requesting Net 30)
- [ ] Deposit collected (if prepay terms)
- [ ] Donovan Weber approval (if non-standard terms requested)

## Post-First-Order Follow-Up (within 30 days of first delivery)

- [ ] Customer satisfaction check (PM call or email)
- [ ] Credit application follow-up (if Net 30 requested)
- [ ] Update Account status from "Onboarding" to "Active"
- [ ] Build initial customer profile (pmlord-customer-profile)
- [ ] Note any preferences, requirements, or relationship insights

## Documentation Requirements

- [ ] Onboarding gate record saved to `~/.pmlord/customers/{CUSTOMER_ID}/`
- [ ] Legitimacy check results documented
- [ ] Credit decision documented with rationale
- [ ] PM assignment documented with rationale
- [ ] Any escalation decisions documented
