# Handoff Checklist

Every handoff -- vacation, permanent, or shared -- must include the following information. This checklist ensures nothing falls through the cracks.

## 1. Active Orders

For each active order, provide:

- [ ] Order ID and quote number
- [ ] Customer name and contact (name, email, phone, preferred method)
- [ ] Part description, part number, quantity
- [ ] Current stage (RFQ, quoting, ordered, manufacturing, shipping, inspection, delivered)
- [ ] Current status within stage (e.g., "manufacturing -- 60% complete, expecting T1 samples March 20")
- [ ] Manufacturing partner and partner contact
- [ ] Next milestone and date
- [ ] All upcoming deadlines within the handoff window
- [ ] Open action items (who needs to do what)
- [ ] Financial summary (order value, margin, payment status)
- [ ] Any change orders in progress or pending
- [ ] Shipping status and tracking numbers (if in transit)

## 2. Open RFQs

For each open RFQ, provide:

- [ ] RFQ ID
- [ ] Customer name and contact
- [ ] Part description and requirements summary
- [ ] Qualification status (qualified, conditional, pending)
- [ ] Quote status (not started, in progress, ready for review, sent to customer)
- [ ] Quote deadline (when customer expects a response)
- [ ] Competitive situation (are they quoting multiple suppliers?)
- [ ] Partner quotes received or pending
- [ ] Key technical challenges or concerns
- [ ] Target price if customer provided one

## 3. Pending Customer Communications

For each pending communication, provide:

- [ ] Customer name and contact
- [ ] Type (email draft, follow-up needed, response awaited)
- [ ] Subject/topic
- [ ] Urgency (when does this need to happen?)
- [ ] Draft content (if applicable)
- [ ] Context (what led to this communication)

## 4. Open Quality Issues

For each NCR or quality issue, provide:

- [ ] NCR ID and related order ID
- [ ] Issue description
- [ ] Current status (open, under investigation, disposition pending, resolved)
- [ ] Current disposition recommendation (use-as-is, rework, scrap, RTV)
- [ ] Customer awareness (have they been notified? what was their reaction?)
- [ ] Partner awareness (has the partner been notified? are they cooperating?)
- [ ] Financial impact (scrap cost, rework cost, replacement cost)
- [ ] Approval status (has disposition been approved per approval matrix?)
- [ ] Root cause analysis status
- [ ] Corrective action status

## 5. Pending Escalations

For each escalation, provide:

- [ ] Escalation type and severity
- [ ] Related order/customer/partner
- [ ] Who escalated and when
- [ ] Current status (open, in progress, awaiting response)
- [ ] Required action and by when
- [ ] Who has been notified

## 6. Customer Relationship Notes

For each active customer, provide:

- [ ] Relationship health (strong, neutral, strained)
- [ ] Key preferences (communication style, frequency, format)
- [ ] Sensitivities (things to avoid, past issues, competitive concerns)
- [ ] Decision-making process (who signs off, how long does it take?)
- [ ] Payment history (on time, slow, issues)
- [ ] Active contracts or agreements (pricing agreements, NDAs, quality agreements)
- [ ] Upcoming meetings or calls scheduled
- [ ] Strategic importance notes

## 7. Partner Context

For each active manufacturing partner, provide:

- [ ] Partner name and primary contact
- [ ] Current orders in production
- [ ] Production status and any delays
- [ ] Quality performance (recent NCRs, scorecard rating)
- [ ] Communication cadence (how often should backup PM check in?)
- [ ] Known issues or concerns
- [ ] Upcoming tooling trials or sample submissions
- [ ] Payment status (invoices pending, PO status)

## 8. Administrative Items

- [ ] Pending invoices to send to customers
- [ ] Pending POs to send to partners
- [ ] Upcoming shipments requiring coordination
- [ ] Insurance or compliance deadlines
- [ ] Scheduled reports or reviews

## Verification

Before finalizing the handoff:

- [ ] Backup PM has reviewed the briefing and has no questions
- [ ] All deadlines within the handoff window are clearly highlighted
- [ ] Emergency contact information is provided for the departing PM
- [ ] Alert routing has been updated
- [ ] CRM notes are current (if applicable)
- [ ] All draft communications are saved and accessible
