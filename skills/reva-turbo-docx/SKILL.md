## Purpose

Convert any REVA-TURBO markdown report to a professionally formatted Word .docx document with Rev A Manufacturing branding. Uses the `report-to-docx.mjs` Node.js script. Called automatically by report-generating skills when `report_format: docx` is set in engine config. Can also be invoked directly by the PM.

## Supported Markdown Constructs

The converter supports the following markdown elements:

| Element | Behavior in DOCX |
|---------|-----------------|
| `# H1` | Title style — Calibri Bold 14pt, centered, top margin |
| `## H2` | Section heading — Calibri Bold 12pt, left-aligned, top spacing |
| `### H3` | Subsection — Calibri Bold 11pt, left-aligned |
| `**bold**` | Bold inline text |
| `*italic*` | Italic inline text |
| `` `code` `` | Inline monospace (Courier New 10pt) |
| Unordered list (`- item`) | Bulleted list with 0.25in indent |
| Ordered list (`1. item`) | Numbered list with 0.25in indent |
| Table | Proper Word table, all columns auto-width, header row bold |
| `---` horizontal rule | Section divider (thin horizontal line) |
| Code block (``` ``` ```) | Monospace block, gray background (Calibri 10pt, shaded) |
| Blockquote (`> text`) | Indented paragraph with left border |
| `{{PLACEHOLDER}}` | **Flagged as missing data** — conversion pauses, PM notified |

## Document Layout

| Setting | Value |
|---------|-------|
| Page size | Letter (8.5 × 11 inches) |
| Margins | 1 inch all sides |
| Body font | Calibri 11pt |
| H1 font | Calibri Bold 14pt |
| H2 font | Calibri Bold 12pt |
| H3 font | Calibri Bold 11pt |
| Header (left) | Company name + report type |
| Header (right) | REVA-TURBO | Date |
| Footer | "Generated by REVA-TURBO \| Confidential \| Page X of Y" |

## Flow

### Step 1 — Resolve Input

Determine the input source:

- If called by another skill, the input file path and report metadata are passed as arguments.
- If invoked directly by PM:

> Which report do you want to convert to .docx?
>
> A) Specify a file path
> B) Convert the most recently generated report
> C) Search for a report by name or order ID

Validate the file exists before proceeding:

```bash
_INPUT="{{INPUT_FILE}}"
[ -f "$_INPUT" ] || { echo "ERROR: File not found: $_INPUT"; exit 1; }
echo "Input file: $_INPUT ($(wc -c < "$_INPUT" | tr -d ' ') bytes)"
```

### Step 2 — Scan for Unfilled Placeholders

Before converting, scan the input for any `{{PLACEHOLDER}}` patterns:

```bash
_PLACEHOLDERS=$(grep -oE '\{\{[A-Z_]+\}\}' "$_INPUT" 2>/dev/null | sort -u)
if [ -n "$_PLACEHOLDERS" ]; then
  echo "PLACEHOLDERS FOUND:"
  echo "$_PLACEHOLDERS"
fi
```

If placeholders are found, **pause and warn the PM:**

> **Unfilled Placeholders Detected**
>
> The following fields are not yet filled in the source document:
>
> {{LIST_OF_PLACEHOLDERS}}
>
> Converting with unfilled placeholders will produce an incomplete document.
>
> A) Fill in the missing fields now, then convert
> B) Convert anyway — I know these fields are incomplete
> C) Cancel conversion

Do NOT proceed to conversion unless PM selects B or after fields are filled.

### Step 3 — Dependency Check

Ensure Node.js and npm dependencies are available:

```bash
# Check Node.js version
node --version 2>/dev/null || { echo "NODE_MISSING"; }

# Check script exists
_DOCX_SCRIPT="$HOME/.claude/skills/reva-turbo/skills/reva-turbo-docx/scripts/report-to-docx.mjs"
[ -f "$_DOCX_SCRIPT" ] || { echo "SCRIPT_MISSING: $_DOCX_SCRIPT"; }

# Check npm dependencies
cd "$(dirname "$_DOCX_SCRIPT")" && npm ls 2>/dev/null | head -5 || echo "DEPS_CHECK_FAILED"
```

**If Node.js is not installed:**

> Node.js is required but not installed. Install it with:
>
> - **macOS:** `brew install node` (requires Homebrew) or download from nodejs.org
> - **Windows:** Download installer from nodejs.org (v18 or higher)
> - **Linux:** `curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash - && sudo apt-get install -y nodejs`
>
> After installing Node.js, re-run this command.

**If npm dependencies are missing:**

```bash
cd "$HOME/.claude/skills/reva-turbo/skills/reva-turbo-docx/scripts" && npm install
```

### Step 4 — Convert

Resolve output directory and filename, then run the script:

```bash
_DOCX_SCRIPT="$HOME/.claude/skills/reva-turbo/skills/reva-turbo-docx/scripts/report-to-docx.mjs"
_OUTPUT_DIR=$(~/.claude/skills/reva-turbo/bin/reva-turbo-config get report_output_dir 2>/dev/null || echo "$HOME/Documents/REVA-TURBO-Reports")
mkdir -p "$_OUTPUT_DIR"

node "$_DOCX_SCRIPT" \
  --input "{{INPUT_FILE}}" \
  --output "$_OUTPUT_DIR/{{FILENAME}}.docx" \
  --title "{{TITLE}}" \
  --company "{{COMPANY_NAME}}"
```

**If the script errors:**

> Conversion failed with error:
> ```
> {{RAW_ERROR}}
> ```
>
> A) Troubleshoot — show me what went wrong
> B) Save as .md fallback instead — I'll convert manually later
> C) Cancel

If PM selects B, copy the source markdown to the output directory with a `.md` extension and note that .docx conversion failed.

### Step 5 — Confirm Output

After successful conversion:

```bash
_OUTPUT_PATH="$_OUTPUT_DIR/{{FILENAME}}.docx"
_FILE_SIZE=$(ls -lh "$_OUTPUT_PATH" 2>/dev/null | awk '{print $5}')
_CHAR_COUNT=$(wc -c < "{{INPUT_FILE}}" 2>/dev/null | tr -d ' ')
_PAGE_ESTIMATE=$(( _CHAR_COUNT / 3000 + 1 ))
echo "Output: $_OUTPUT_PATH"
echo "Size: $_FILE_SIZE"
echo "Estimated pages: $_PAGE_ESTIMATE"
```

Report to PM:

> **Conversion Complete**
>
> File: `{{OUTPUT_PATH}}`
> Size: {{FILE_SIZE}}
> Estimated pages: {{PAGE_ESTIMATE}}
>
> The .docx file is ready. Open it in Word or share it directly.

### Step 6 — Log

```bash
echo '{"ts":"'$(date -u +%Y-%m-%dT%H:%M:%SZ)'","action":"docx_convert","input":"{{INPUT_FILE}}","output":"{{OUTPUT_FILE}}","pages_est":{{PAGE_ESTIMATE}},"pm":"{{PM_NAME}}"}' >> ~/.reva-turbo/state/docx-log.jsonl 2>/dev/null || true
```

## File Naming Convention

All REVA-TURBO reports must follow this naming pattern:

```
REVA-TURBO-{Type}-{YYYY-MM-DD}-{ShortName}.docx
```

Examples:
- `REVA-TURBO-WeeklyReport-2026-04-04-Week14.docx`
- `REVA-TURBO-Inspection-2026-04-01-PN4820.docx`
- `REVA-TURBO-AuditReport-2026-03-31-ORD-2026-0142.docx`
- `REVA-TURBO-NCR-2026-03-28-Shenzhen-Precision.docx`

## Rules

1. **No data loss.** The .docx must contain all content from the markdown source. Nothing is silently omitted.
2. **No silent placeholders.** Any `{{PLACEHOLDER}}` in the input must trigger a PM warning before conversion proceeds.
3. **Rev A branding.** Every document includes the company header, REVA-TURBO footer, and page numbers.
4. **Table formatting.** Markdown tables render as proper Word tables with borders and a bold header row.
5. **Fallback to .md.** If conversion fails and PM chooses fallback, save the .md file to the output directory and note the failure.
6. **Report output dir.** Always use the configured `report_output_dir`. Never write to arbitrary paths.
