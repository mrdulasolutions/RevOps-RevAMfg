# REVA-TURBO Data Carryover Schema

This document defines the explicit field carryover map for each lifecycle transition in the REVA-TURBO PM workflow. For each transition, it lists the fields passed to the next stage, the data type, the source skill, whether the field is required or optional, and where the field is stored in state.

All state is persisted in `~/.reva-turbo/state/workflow-state.jsonl` (one JSON line per stage transition event) unless otherwise noted. Fields that are stored in separate state files are noted in the "State Location" column.

---

## Transition 1: rfq-intake → rfq-qualify

**Source skill:** reva-turbo-rfq-intake
**Target skill:** reva-turbo-rfq-qualify
**State file:** `~/.reva-turbo/state/workflow-state.jsonl`

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `rfq_id` | string | Yes | Generated by rfq-intake: `RFQ-YYYY-NNNN` |
| `customer_name` | string | Yes | Contact name of the person who submitted the RFQ |
| `customer_email` | string | Yes | Contact email |
| `customer_company` | string | Yes | Company name — used as the primary customer identifier |
| `part_number` | string | Yes | Rev A PN if assigned; customer PN if not yet assigned |
| `part_description` | string | Yes | Human-readable part description |
| `quantity` | integer | Yes | Order quantity |
| `material` | string | Yes | Primary material (e.g. "6061 aluminum", "304 stainless") |
| `application` | string | Yes | End-use application — required for export compliance pre-screen |
| `delivery_date_requested` | date (YYYY-MM-DD) | Yes | Customer's requested delivery date |
| `attachments_list` | string[] | Optional | Filenames of drawings, specs, or other documents attached to the RFQ |
| `source_channel` | string | Yes | How the RFQ arrived: `email` / `website` / `phone` / `crm` |
| `intake_ts` | ISO 8601 UTC | Yes | Timestamp when rfq-intake ran |
| `raw_rfq_text` | string | Optional | Original email text or form submission — for reference |

## Transition 2: rfq-qualify → export-compliance (or customer-gate)

**Source skill:** reva-turbo-rfq-qualify
**Target skill:** reva-turbo-export-compliance (returning customer) or reva-turbo-customer-gate (new customer)
**State file:** `~/.reva-turbo/state/workflow-state.jsonl`

All fields from Transition 1, plus:

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `qualify_score` | integer (0–100) | Yes | Qualification score from rfq-qualify |
| `qualify_outcome` | string | Yes | `pass` / `conditional` / `reject` |
| `customer_type` | string | Yes | `new` / `returning` — drives routing to customer-gate or export-compliance |
| `credit_status` | string | Yes | `approved` / `pending` / `hold` — from customer profile or CRM |
| `capability_match` | string | Yes | `yes` / `no` / `conditional` — can Rev A make this part? |
| `capacity_available` | boolean | Yes | Does Rev A have capacity for this order in the timeframe? |
| `qualify_notes` | string | Optional | PM notes on the qualification decision |

## Transition 3: customer-gate → export-compliance

**Source skill:** reva-turbo-customer-gate
**Target skill:** reva-turbo-export-compliance
**State file:** `~/.reva-turbo/state/workflow-state.jsonl`
**Customer profile:** `~/.reva-turbo/state/customers/{{customer-slug}}.yaml`

All fields from Transition 2, plus:

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `customer_slug` | string | Yes | URL-safe identifier generated at customer-gate (e.g. `acme-corp`) |
| `customer_profile_created` | boolean | Yes | Confirms profile file was written |
| `nda_status` | string | Yes | `signed` / `pending` / `none` |
| `credit_terms` | string | Yes | `net30` / `prepay` / `cod` / other |
| `credit_approved` | boolean | Yes | Credit line approved by finance |
| `onboarding_ts` | ISO 8601 UTC | Yes | Timestamp when customer-gate completed |

## Transition 4: export-compliance → rfq-quote

**Source skill:** reva-turbo-export-compliance
**Target skill:** reva-turbo-rfq-quote
**State file:** `~/.reva-turbo/state/workflow-state.jsonl`
**ExChek report:** path stored in `exchek_report_path`

All fields from Transition 3 (or Transition 2 for returning customers), plus:

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `eccn` | string | Yes | ECCN classification (e.g. `EAR99`, `2B001.b`) |
| `itar_status` | string | Yes | `controlled` / `not_controlled` |
| `sanctions_clear` | boolean | Yes | `true` = customer and country cleared against SDN/denied party lists |
| `license_status` | string | Yes | `no_license_required` / `license_required` / `license_granted` / `denied` |
| `exchek_report_path` | string | Yes | Absolute path to the ExChek compliance report file |
| `compliance_cleared` | boolean | Yes | `true` only if all screens pass and no license is required (or license is granted) |
| `compliance_ts` | ISO 8601 UTC | Yes | Timestamp of compliance determination |

**Note:** If `compliance_cleared` is `false`, the order DOES NOT advance to rfq-quote. It is held pending escalation resolution.

## Transition 5: rfq-quote → partner-select

**Source skill:** reva-turbo-rfq-quote
**Target skill:** reva-turbo-partner-select (after customer accepts quote)
**State file:** `~/.reva-turbo/state/workflow-state.jsonl`
**Quote document:** path stored in `quote_doc_path`

All fields from Transition 4, plus:

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `quote_id` | string | Yes | Quote identifier: `QTE-YYYY-NNNN` |
| `unit_price` | decimal | Yes | Per-unit customer price (USD) |
| `total_price` | decimal | Yes | Total customer price = unit_price × quantity |
| `estimated_cost_breakdown` | object | Yes | `{partner_quote, tooling, materials, shipping_intl, duties_tariffs, overhead}` all in USD |
| `target_margin` | decimal | Yes | Target gross margin percentage (e.g. `0.35` = 35%) |
| `lead_time_days` | integer | Yes | Estimated manufacturing + shipping lead time in calendar days |
| `quote_expiry` | date (YYYY-MM-DD) | Yes | Date quote expires (default: 30 days from issue) |
| `payment_terms` | string | Yes | Payment terms offered to customer (e.g. `net30`) |
| `quote_doc_path` | string | Yes | Absolute path to the generated quote document |
| `quote_sent_ts` | ISO 8601 UTC | Yes | Timestamp when quote was sent to customer |

**Note:** Transition to partner-select only occurs after customer accepts (PO received). The `quote_sent_ts` is set when the quote email is sent. A separate `po_received_ts` is logged when the customer PO arrives.

## Transition 6: partner-select → china-package

**Source skill:** reva-turbo-partner-select
**Target skill:** reva-turbo-china-package
**State file:** `~/.reva-turbo/state/workflow-state.jsonl`
**Selection detail:** also logged to `~/.reva-turbo/state/audit-trail.jsonl`

All fields from Transition 5, plus:

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `selected_partner` | string | Yes | Partner name (matches `name` field in partners.yaml) |
| `partner_slug` | string | Yes | Partner slug (matches `slug` field in partners.yaml) |
| `partner_location` | string | Yes | City, Country (e.g. "Shenzhen, CN") |
| `partner_score` | integer (0–100) | Yes | Score from partner-select capability scoring |
| `selection_rationale` | string | Yes | 3-sentence written rationale for selection — logged to audit trail |
| `partner_ip_status` | string | Yes | `nnn` / `nda` / `limited` / `none` — IP agreement status |
| `alternatives_considered` | string[] | Yes | List of all other partners scored (not selected) — for audit |

## Transition 7: china-package → china-track

**Source skill:** reva-turbo-china-package
**Target skill:** reva-turbo-china-track
**State file:** `~/.reva-turbo/state/workflow-state.jsonl`
**Package contents:** saved to `~/.reva-turbo/state/packages/{{rfq_id}}/`

All fields from Transition 6, plus:

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `package_sent_date` | date (YYYY-MM-DD) | Yes | Date manufacturing package was sent to partner |
| `package_contents` | string[] | Yes | List of filenames in the manufacturing package (drawings, specs, instructions) |
| `drawing_revision` | string | Yes | Drawing revision level sent (e.g. "Rev C") |
| `special_instructions` | string | Optional | Any special handling, tolerance callouts, or surface finish notes beyond the drawing |
| `tooling_required` | boolean | Yes | Does this order require new tooling? |
| `tooling_approved` | boolean | Yes | Has tooling been reviewed and approved by PM? |
| `target_ship_date` | date (YYYY-MM-DD) | Yes | Expected ship date from partner factory |
| `partner_po_number` | string | Yes | Purchase order number issued to the partner |

## Transition 8: china-track → import-compliance

**Source skill:** reva-turbo-china-track
**Target skill:** reva-turbo-import-compliance
**State file:** `~/.reva-turbo/state/workflow-state.jsonl`
**Tracking log:** `~/.reva-turbo/shipments/shipment-log.jsonl`

All fields from Transition 7, plus:

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `ship_date` | date (YYYY-MM-DD) | Yes | Actual date goods shipped from partner factory |
| `carrier` | string | Yes | Carrier name (e.g. "DHL", "Flexport", "Expeditors") |
| `tracking_number` | string | Yes | Carrier tracking / AWB / B/L number |
| `port_of_export` | string | Yes | Port where goods departed China (e.g. "Shenzhen Yantian", "Shanghai Pudong") |
| `eta_port_of_entry` | date (YYYY-MM-DD) | Yes | Estimated arrival at US port of entry |
| `invoice_value` | decimal | Yes | Invoice value in USD (actual commercial invoice value) |
| `invoice_currency` | string | Yes | Currency of invoice (`USD` / `CNY`) |
| `packing_list_confirmed` | boolean | Yes | Packing list reviewed and confirmed to match PO |
| `milestone_log` | object[] | Optional | Array of manufacturing milestones with actual vs expected dates |

## Transition 9: import-compliance → logistics

**Source skill:** reva-turbo-import-compliance
**Target skill:** reva-turbo-logistics
**State file:** `~/.reva-turbo/state/workflow-state.jsonl`
**Customs docs:** stored in `~/.reva-turbo/shipments/customs/{{rfq_id}}/`

All fields from Transition 8, plus:

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `hts_codes` | string[] | Yes | HTS classification codes for all line items |
| `duty_amount` | decimal | Yes | Total estimated duty in USD |
| `effective_duty_rate` | decimal | Yes | Blended effective duty rate as a percentage |
| `section_301_applies` | boolean | Yes | Whether Section 301 China tariffs apply |
| `customs_docs_status` | string | Yes | `complete` / `pending` / `exception` |
| `isf_filed` | boolean | Yes | ISF filing confirmed (ocean shipments) |
| `flow_type` | string | Yes | `direct` (China→Customer) or `i_and_f` (Inspect & Forward) |
| `compliance_cleared` | boolean | Yes | `true` = import compliance cleared, ok to proceed to logistics |
| `compliance_ts` | ISO 8601 UTC | Yes | Timestamp of import compliance determination |

## Transition 10: logistics → order-track (closeout)

**Source skill:** reva-turbo-logistics (and downstream skills in I&F path)
**Target skill:** reva-turbo-order-track → reva-turbo-profit → reva-turbo-audit-trail
**State file:** `~/.reva-turbo/state/workflow-state.jsonl`
**Shipment log:** `~/.reva-turbo/shipments/shipment-log.jsonl`

All fields from Transition 9, plus:

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `routing_type` | string | Yes | `direct` or `i_and_f` — final routing decision |
| `final_tracking_number` | string | Yes | Domestic or final delivery tracking number |
| `delivery_date_estimated` | date (YYYY-MM-DD) | Yes | Estimated delivery to customer |
| `delivery_date_actual` | date (YYYY-MM-DD) | Yes | Actual delivery date (updated when confirmed) |
| `customs_entry_number` | string | Yes | US CBP entry number |
| `rev_a_coc_path` | string | Optional | Path to Rev A Certificate of Conformance (I&F flow only) |
| `inspection_report_path` | string | Optional | Path to inspection report (I&F flow only) |
| `inspection_disposition` | string | Optional | `accept` / `reject` / `hold` (I&F flow only) |
| `total_cost_actual` | decimal | Yes | Total actual landed cost in USD (from reva-turbo-cost-tracking) |
| `total_revenue` | decimal | Yes | Total customer invoice amount in USD |
| `gross_margin` | decimal | Yes | Calculated gross margin percentage |

## State File Summary

| State File | Contains |
|-----------|---------|
| `~/.reva-turbo/state/workflow-state.jsonl` | One entry per stage transition; carries all carryover fields forward |
| `~/.reva-turbo/state/audit-trail.jsonl` | Compliance, quality, override, and escalation decisions |
| `~/.reva-turbo/state/cost-ledger.jsonl` | Estimated vs actual costs per order per category |
| `~/.reva-turbo/state/inspection-log.jsonl` | Inspection events and dispositions |
| `~/.reva-turbo/state/partner-history.jsonl` | Partner events — orders, score changes, NCRs |
| `~/.reva-turbo/shipments/shipment-log.jsonl` | Shipment events — booking, tracking, exceptions |
| `~/.reva-turbo/config/partners.yaml` | Authoritative partner master record |
| `~/.reva-turbo/state/customers/{{slug}}.yaml` | Customer profiles (one file per customer) |
| `~/.reva-turbo/state/packages/{{rfq_id}}/` | Manufacturing package contents per order |
| `~/.reva-turbo/shipments/customs/{{rfq_id}}/` | Customs documentation per shipment |
