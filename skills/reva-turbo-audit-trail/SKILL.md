## Preamble (run first)

```bash
mkdir -p ~/.reva-turbo/analytics
echo '{"skill":"reva-turbo-audit-trail","ts":"'$(date -u +%Y-%m-%dT%H:%M:%SZ)'"}' >> ~/.reva-turbo/analytics/skill-usage.jsonl 2>/dev/null || true
```

## Purpose

Maintain a complete, immutable audit trail for every significant PM decision and action at Rev A Manufacturing. The audit trail is append-only — entries are never modified or deleted. This provides accountability, traceability, and compliance documentation.

## Flow

### Step 1: Determine Action

> What audit trail action do you need?
> A) Log a new audit entry (manual)
> B) Query the audit trail (search/filter)
> C) Generate an audit report for a specific order
> D) Generate an audit report for a date range
> E) Verify audit trail integrity

### Step 2: Log Entry (Manual)

For manual audit entries, collect:

- **Action Type:** {{ACTION_TYPE}} (decision, approval, override, communication, escalation, correction)
- **Related PO/RFQ:** {{REFERENCE_ID}}
- **Customer:** {{CUSTOMER_COMPANY}}
- **Description:** {{DESCRIPTION}}
- **Justification:** {{JUSTIFICATION}} (required for overrides and corrections)
- **PM:** {{PM_NAME}}

Write the entry (append-only):

```bash
mkdir -p ~/.reva-turbo/state
echo '{"ts":"'$(date -u +%Y-%m-%dT%H:%M:%SZ)'","type":"{{ACTION_TYPE}}","ref":"{{REFERENCE_ID}}","customer":"{{CUSTOMER_COMPANY}}","action":"{{DESCRIPTION}}","justification":"{{JUSTIFICATION}}","pm":"{{PM_NAME}}","skill":"manual"}' >> ~/.reva-turbo/state/audit-trail.jsonl
```

### Step 3: Query Audit Trail

Support the following query types:

1. **By PO/RFQ number** — All entries for a specific order
2. **By customer** — All entries for a specific customer
3. **By PM** — All entries by a specific PM
4. **By date range** — All entries within a time window
5. **By action type** — All entries of a specific type
6. **By skill** — All entries generated by a specific REVA-TURBO skill

```bash
# Example: Query by PO number
grep '"ref":"{{PO_NUMBER}}"' ~/.reva-turbo/state/audit-trail.jsonl 2>/dev/null

# Example: Query by customer
grep '"customer":"{{CUSTOMER_COMPANY}}"' ~/.reva-turbo/state/audit-trail.jsonl 2>/dev/null

# Example: Query by date (partial match on timestamp)
grep '"ts":"{{DATE_PREFIX}}' ~/.reva-turbo/state/audit-trail.jsonl 2>/dev/null
```

### Step 4: Generate Audit Report

Use `templates/Audit Trail Entry.md` to format a human-readable audit report for the queried entries.

### Step 5: Integrity Verification

Count entries and verify the audit trail file has not been truncated:

```bash
wc -l ~/.reva-turbo/state/audit-trail.jsonl 2>/dev/null
ls -la ~/.reva-turbo/state/audit-trail.jsonl 2>/dev/null
```

Compare against the last known entry count from the integrity log:

```bash
echo '{"ts":"'$(date -u +%Y-%m-%dT%H:%M:%SZ)'","action":"integrity_check","entries":'$(wc -l < ~/.reva-turbo/state/audit-trail.jsonl 2>/dev/null || echo 0)',"status":"verified"}' >> ~/.reva-turbo/state/audit-integrity.jsonl
```

## Audit Entry Schema

Every audit trail entry must contain:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `ts` | ISO 8601 UTC | Yes | Timestamp of the action |
| `type` | string | Yes | Action type (decision, approval, override, communication, escalation, correction, system) |
| `ref` | string | Yes | Related PO or RFQ number |
| `customer` | string | Yes | Customer company name |
| `action` | string | Yes | Description of what was done |
| `justification` | string | For overrides/corrections | Why the action was taken |
| `pm` | string | Yes | PM who took the action |
| `skill` | string | Yes | REVA-TURBO skill that generated the entry |
| `detail` | string | No | Additional detail or context |

## Retention Policy

All audit trail records are subject to the following minimum retention periods. Records are **never deleted** — they are flagged for archival after the retention period expires. The `archive_before_date` flag in audit queries marks records eligible for archival.

| Record Type | Retention Period | Regulatory Basis |
|-------------|-----------------|-----------------|
| Standard workflow events (stage transitions, partner comms) | 7 years | EAR export compliance recordkeeping requirement (15 CFR 762) |
| Export compliance gate decisions | 10 years | EAR 15 CFR 762 — license determination records |
| Import compliance gate decisions | 10 years | CBP recordkeeping requirements (19 CFR 163) |
| Customer communications | 5 years | Standard commercial practice |
| Quality gate records (G1–G4 pass/fail) | 7 years | ISO 9001 quality management recordkeeping |
| NCR (Non-Conformance Report) records | 10 years | ISO 9001 + customer contract requirements |
| Escalation records | 7 years | Internal policy |
| Partner selection decisions | 7 years | EAR recordkeeping |
| Financial / pricing decisions | 7 years | Standard commercial + tax recordkeeping |

**Archival implementation:**
- Records past their retention period are flagged with `"archive_eligible": true` in audit queries.
- They are **not deleted** — they are moved to `~/.reva-turbo/state/audit-archive.jsonl`.
- A PM must authorize archival — it is never automatic.
- Archival log is separate from the main audit trail.

```bash
# Flag records older than retention period (example: 7 years)
_CUTOFF=$(date -u -d "-7 years" +%Y-%m-%dT%H:%M:%SZ 2>/dev/null || date -u -v-7y +%Y-%m-%dT%H:%M:%SZ 2>/dev/null || echo "CHECK_DATE_MANUALLY")
echo "Records before $_CUTOFF are eligible for archival review."
grep "\"ts\":\"" ~/.reva-turbo/state/audit-trail.jsonl 2>/dev/null | awk -F'"ts":"' '{print $2}' | cut -c1-20 | sort | head -5
```

## Compliance Audit Mode

**Command:** `/reva-turbo:reva-turbo-audit-trail compliance-audit`

Optionally: `/reva-turbo:reva-turbo-audit-trail compliance-audit --from YYYY-MM-DD --to YYYY-MM-DD`

This mode produces a filtered compliance audit report formatted for ISO 9001 / EAR audit presentation.

```bash
_FROM="${1:-$(date -u -d "-12 months" +%Y-%m-%d 2>/dev/null || date -u -v-12m +%Y-%m-%d)}"
_TO="${2:-$(date -u +%Y-%m-%d)}"
echo "Compliance audit: $_FROM to $_TO"

# Export compliance decisions
echo "=== EXPORT COMPLIANCE DECISIONS ==="
grep '"type":"compliance_decision"' ~/.reva-turbo/state/audit-trail.jsonl 2>/dev/null | \
  awk -v from="$_FROM" -v to="$_TO" '$0 >= from && $0 <= to' | head -100

# Quality gate decisions
echo "=== QUALITY GATE DECISIONS ==="
grep '"skill":"reva-turbo-quality-gate"' ~/.reva-turbo/state/audit-trail.jsonl 2>/dev/null | head -100

# Escalation events
echo "=== ESCALATION EVENTS ==="
grep '"type":"escalation"' ~/.reva-turbo/state/audit-trail.jsonl 2>/dev/null | head -100
```

## Rules

1. **Append-only.** Never modify or delete existing entries.
2. **Every decision gets logged.** Stage transitions, routing overrides, NCR dispositions, escalations, quote approvals — all must be logged.
3. **Justification is mandatory for overrides.** Any time a PM overrides a system recommendation, the justification field must be populated.
4. **Timestamps are UTC.** All timestamps use ISO 8601 UTC format.
5. **No PII in audit entries.** Use customer company names and PO numbers, not personal contact details.
6. **Retention is mandatory.** Records are never deleted. Only archival (with PM authorization) after retention period is permitted.
7. **Compliance audit is always available.** `/reva-turbo:reva-turbo-audit-trail compliance-audit` must be runnable at any time for external auditors.
